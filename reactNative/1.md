## 使用库 Expo Open OTA

> 项目地址[https://axelmarciano.github.io/expo-open-ota](https://axelmarciano.github.io/expo-open-ota)
>
> 开源，与expo深度整合，使用说明见官网

## 大致使用方式及问题排除

> 只是大致步骤，具体使用方法见官网文档
>
> 1.获取expo Access Token [https://expo.dev/accounts/swordfly/settings/access-tokens](https://expo.dev/accounts/swordfly/settings/access-tokens)
>
> 2.Project ID Run `eas project:info` in your Expo project
>
> 3.生成签名证书 npx eoas generate-certs in your Expo project
>
> - 此命令生成三个证书文件位于 certs/
> - public-key.pem，private-key.pem(服务器用于签名和验证更新)。
> - certificate.pem(用于app更新验证)**_注意：当执行eas build时需要上传此文件请在.gitignore中去除此目录或扩展名，eas build时会报错_**
>
> 4.服务端设置。
>
> ```bash
> #docker-compose.yml文件 
> services:
>   expo-ota:
>     image: ghcr.io/axelmarciano/expo-open-ota:latest
>     network_mode: host #如果使用podman-compose运行，添加此项配置，删除ports:"${PORT}:3000"
>     container_name: expo-ota
>     restart: unless-stopped
>     ports:
>       - "${PORT}:3000"
>     environment:
>       - BASE_URL=${BASE_URL}
>       - EXPO_ACCESS_TOKEN=${EXPO_ACCESS_TOKEN}
>       - EXPO_APP_ID=${EXPO_APP_ID}
>       - JWT_SECRET=${JWT_SECRET}
>       - STORAGE_MODE=${STORAGE_MODE}
>       - KEYS_STORAGE_TYPE=${KEYS_STORAGE_TYPE}
>       - PUBLIC_LOCAL_EXPO_KEY_PATH=${PUBLIC_LOCAL_EXPO_KEY_PATH}
>       - PRIVATE_LOCAL_EXPO_KEY_PATH=${PRIVATE_LOCAL_EXPO_KEY_PATH}
>       - CACHE_MODE=${CACHE_MODE}
>       - USE_DASHBOARD=${USE_DASHBOARD}
>       - ADMIN_PASSWORD=${ADMIN_PASSWORD}
>     volumes:
>       - ./certs:/keys:ro
>       - ./updates:/app/updates
> #.env文件内容：
> # 端口
> PORT=3000
>
> # 你的域名（必须 HTTPS）
> BASE_URL=https://ota.xinde.org
>
> # Expo 项目信息
> EXPO_ACCESS_TOKEN=expo access token
> EXPO_APP_ID=expo project id
>
> # 安全密钥
> JWT_SECRET= 32位秘钥  #openssl rand -base64 32 命令生成
> ADMIN_PASSWORD=admin password
>
> # 存储配置
> STORAGE_MODE=local
> KEYS_STORAGE_TYPE=local
> PUBLIC_LOCAL_EXPO_KEY_PATH=/keys/public-key.pem
> PRIVATE_LOCAL_EXPO_KEY_PATH=/keys/private-key.pem
> CACHE_MODE=local
> USE_DASHBOARD=true
> #以上文件配置好后在哪前目录运行
> #docker compose up -d 起动docker容器
> # docker compose down 是结束容器
> # docker常见命令
> #	删除容器 docker rm -f expo-ota
> #	查看容器运行状态 docker ps
> #	查看运行中容器 docker logs expo-ota
> #=================以下为minio对像存储配置
> version: '3.8'
>
> services:
>   minio:
>     image: minio/minio:latest
>     container_name: minio
>     restart: always
>     ports:
>       - "9000:9000"   # S3 API
>       - "9001:9001"   # Web控制台
>     volumes:
>       - /data/minio/data:/data          # 数据盘
>       - /data/minio/config:/root/.minio # 配置
>     environment:
>       MINIO_ROOT_USER: admin        # 账号（≥5位）
>       MINIO_ROOT_PASSWORD: YourStrongPass123! # 密码（≥8位，改复杂！）
>     command: server /data --console-address ":9001"
> ```
>
> 5.配置EXPO App
>
> ```bash
> npx eoas init
> # 执行完后会生成app.config.js配置文件。执行完成后需要执行一次eas build重新打包。如app.config.js中使用环境变量，打包时需指定。
> ```
>
> 6. 创建发布通道，如已发布过可跳过此步。
> 7. 发布更新
>
> ```
> npx eoas publish --branch production
> ```
